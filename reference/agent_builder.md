# Спецификация: Редактор агента
**Версия:** 1.0  
**Статус:** Драфт

---

## 1. Концепция и принципы

Редактор агента — страница для создания и редактирования конфигурации агента. Все компоненты агента доступны одновременно на одном экране, без обязательной последовательности заполнения.

**Ключевые принципы:**

**Единая страница с секциями.** Wizard не используется — порядок заполнения не важен. Пользователь видит все компоненты сразу и работает с ними в любом порядке. Незаполненные секции показывают плейсхолдер и кнопку «Добавить».

**Конфигурация через ссылки.** Агент не хранит компоненты внутри себя — он ссылается на существующие объекты: навыки, знания, инструменты. Редактор показывает эту модель явно: пользователь выбирает объекты из каталогов.

**Версионность как основа.** Каждое сохранение создаёт новую версию. Пользователь всегда видит текущую версию и может просмотреть историю, сравнить или откатиться.

**Тест как часть редактирования.** Встроенный чат доступен прямо в редакторе для проверки текущей конфигурации без выхода на отдельную страницу.

## 2. Точка входа

### 2.1 Стартовый экран

При создании нового агента открывается стартовый экран — он предшествует редактору и закрывается после выбора пути.

**Два варианта входа:**

**Описать текстом** — текстовое поле «Опишите вашего агента». Система генерирует черновик конфигурации: заполняет системную инструкцию, предлагает подходящие инструменты и навыки. Пользователь попадает в редактор с готовым черновиком.

**Выбрать шаблон** — библиотека шаблонов с поиском. Карточка шаблона содержит: название, описание задачи, иконки задействованных инструментов. Выбор шаблона открывает редактор с предзаполненной конфигурацией.

При **редактировании** существующего агента стартовый экран пропускается — редактор открывается сразу с текущей конфигурацией.

## 3. Структура интерфейса

```
┌────────────────────────────────────────────────────────────────────┐
│  ← Закупки / Юридический ассистент    v3 ↓    [Сохранить] [Тест]  │  ← Шапка
├───────────────────────────┬────────────────────────────────────────┤
│                           │                                        │
│  СИСТЕМНАЯ ИНСТРУКЦИЯ     │                                        │
│  ─────────────────────    │   ЧАТ-ПРЕВЬЮ                          │
│  НАВЫКИ                   │   (встроенное тестирование)            │
│  ─────────────────────    │                                        │
│  ЗНАНИЯ КОМПАНИИ          │                                        │
│  ─────────────────────    │                                        │
│  ИНСТРУМЕНТЫ              │                                        │
│  ─────────────────────    │                                        │
│  ТРИГГЕРЫ                 │                                        │
│                           │                                        │
└───────────────────────────┴────────────────────────────────────────┘
```

**Левая колонка (~60%)** — конфигурация агента, секции с вертикальной прокруткой.

**Правая колонка (~40%)** — встроенный чат-превью для тестирования. Тестирует актуальную сохранённую версию. Можно свернуть, освободив всю ширину под конфигурацию.

## 4. Шапка редактора

| Элемент | Описание |
|---|---|
| Хлебная крошка | Пространство / Название агента |
| Селектор версии | «v3 ↓» — кнопка, открывающая панель версий |
| Кнопка «Сохранить» | Создаёт новую версию. Активна при наличии несохранённых изменений |
| Индикатор изменений | Точка рядом с «Сохранить» при несохранённых правках |
| Кнопка «Тест» | Открывает/закрывает чат-превью справа |

## 5. Версии агента

### 5.1 Селектор версий

Кнопка в шапке показывает номер текущей версии. При нажатии открывается выпадающая панель версий.

**Панель версий:**

```
┌─────────────────────────────────────┐
│  История версий                 [✕] │
├─────────────────────────────────────┤
│  ● v4  Сегодня, 14:32  [Текущая]   │
│    v3  Сегодня, 11:05               │
│    v2  Вчера, 18:44                 │
│    v1  12 апр, 10:00  [Исходная]   │
├─────────────────────────────────────┤
│  [Сравнить версии]                  │
└─────────────────────────────────────┘
```

Каждая строка версии: номер, дата и время сохранения, метка «Текущая» / «Исходная». При наведении появляются кнопки: «Просмотреть» и «Восстановить».

### 5.2 Поведение версий

**Сохранение** — каждое нажатие «Сохранить» создаёт новую пронумерованную версию. Версия не создаётся автоматически — только явным действием пользователя.

**Просмотр версии** — открывает конфигурацию этой версии в режиме «только чтение». Шапка показывает: «Просмотр v2 · [Вернуться к текущей] [Восстановить]».

**Восстановление** — создаёт новую версию на основе выбранной. Пример: восстановление v2 создаёт v5 с конфигурацией v2. Предыдущие версии не удаляются.

**Сравнение** — кнопка «Сравнить версии» открывает diff-вид: две колонки с выбором версий, изменённые поля подсвечены.

### 5.3 Индикация несохранённых изменений

При наличии несохранённых правок: точка-индикатор рядом с кнопкой «Сохранить», плашка «Есть несохранённые изменения» в шапке. При попытке закрыть страницу — диалог подтверждения.

## 6. Секции конфигурации

Каждая секция — карточка с заголовком, действиями и телом. Незаполненная секция показывает серый плейсхолдер. Секции нельзя переставлять — их порядок отражает логическую структуру агента.

### 6.1 Системная инструкция

**Заголовок:** «Системная инструкция» + кнопка «Редактировать»

**Тело:** первые 3–4 строки текста инструкции, обрезанные с `...`

**Плейсхолдер:** «Опишите как агент должен себя вести, какую роль выполнять и какие ограничения соблюдать»

**Редактирование:** кнопка «Редактировать» → попап с markdown-редактором. Поддерживает переменные в формате `{{variable_name}}` с автодополнением.

---

### 6.2 Навыки

**Заголовок:** «Навыки» + кнопки «+ Добавить»

**Тело:** список подключённых навыков. Каждый элемент: название навыка, уровень (компания / пространство), кнопка удаления.

**Плейсхолдер:** «Добавьте навыки — готовые сценарии поведения агента в конкретных ситуациях»

**Добавление:** попап с каталогом навыков, разделённых на две группы: навыки компании и навыки пространства. Поиск по названию.

---

### 6.3 Знания компании

**Заголовок:** «Знания компании» + кнопка «+ Подключить»

**Тело:** список подключённых баз знаний. Каждый элемент: название базы, количество документов, кнопка отключения.

**Плейсхолдер:** «Подключите базы знаний — агент будет использовать их при ответах»

**Добавление:** попап со списком доступных баз знаний компании. Поиск. При выборе база подключается — агент ссылается на неё, не копирует.

---

### 6.4 Инструменты (MCP)

**Заголовок:** «Инструменты» + кнопки «+ Добавить» и «MCP»

**Тело:** список подключённых инструментов. Каждый элемент: иконка, название, источник, бейдж статуса («Активен» / «Требует настройки» / «Ошибка»), кнопки тест (▷) и удалить.

**Плейсхолдер:** «Добавьте инструменты — внешние системы, которые агент может вызывать»

**Добавление через каталог:** попап с инструментами по категориям (Поиск, Данные, Коммуникации, Код и др.).

**Добавление через MCP:** отдельный попап с формой подключения MCP-сервера: URL, тип авторизации, токен.

**Статус «Требует настройки»** — инструмент доступен в компании, но пользователь не настроил свою учётную запись в целевой системе. Бейдж ведёт на страницу настройки.

---

### 6.5 Триггеры

**Заголовок:** «Триггеры» + кнопка «+ Добавить»

**Тело:** список настроенных триггеров. Каждый элемент: тип триггера, описание условия, статус (активен / приостановлен), кнопки редактировать и удалить.

**Плейсхолдер:** «Добавьте триггеры — условия автоматического запуска агента без участия пользователя»

**Добавление:** попап с выбором типа триггера:

| Тип | Описание |
|---|---|
| По расписанию | Cron-выражение или интервал |
| По событию | Webhook — внешняя система отправляет сигнал |
| По входящему сообщению | Email или очередь сообщений |

После выбора типа — в том же попапе раскрываются поля параметров + кнопка «Сохранить».

---

## 7. Чат-превью (встроенное тестирование)

Правая колонка редактора. Тестирует последнюю **сохранённую** версию агента.

**Шапка чата:** «Тест · v3» — показывает какая версия тестируется.

**Плашка при несохранённых изменениях:** «Вы редактируете v4 — сохраните, чтобы протестировать изменения» + кнопка «Сохранить».

**Поведение:** стандартный чат. Документы из пространства доступны в контексте. История тестовых сообщений не сохраняется при обновлении страницы.

**Кнопка «Очистить»** — сбрасывает историю чата и начинает тестовую сессию заново.

## 8. Попапы редактирования

### 8.1 Текстовый редактор

Используется для системной инструкции и создания нового навыка.

```
┌──────────────────────────────────────────┐
│  Системная инструкция              [✕]   │
├──────────────────────────────────────────┤
│                                          │
│  [Markdown-редактор]                     │
│                                          │
├──────────────────────────────────────────┤
│                     [Отмена]  [Сохранить]│
└──────────────────────────────────────────┘
```

Поддерживает: базовый markdown, переменные `{{variable_name}}` с автодополнением, предпросмотр.

### 8.2 Каталог с поиском

Используется для выбора навыков, инструментов, знаний.

```
┌──────────────────────────────────────────────────┐
│  Добавить инструмент   [🔍 Поиск]          [✕]  │
├──────────────────┬───────────────────────────────┤
│  Все             │  [Карточка]  [Карточка]        │
│  Поиск           │                               │
│  Данные          │  [Карточка]  [Карточка]        │
│  Коммуникации    │                               │
│  Код             │                               │
└──────────────────┴───────────────────────────────┘
```

Левая колонка — категории. Правая — карточки: иконка, название, описание. Клик по карточке добавляет и закрывает попап. При активном поиске левая колонка скрывается, результаты показываются по всем категориям.

## 9. Состояния

| Состояние | Визуальное отображение |
|---|---|
| Не настроено | Серый плейсхолдер-текст в секции |
| Требует действия | Жёлтый бейдж ⚠️ на секции |
| Ошибка | Красный бейдж, красная граница секции |
| Активно | Стандартный вид |
| Сохранение | «Сохранение...» в шапке |
| Сохранено | «Сохранено · v4» в шапке, затем исчезает |
| Несохранённые изменения | Точка у кнопки «Сохранить» |

---

## 10. Вне области текущей спецификации

- Мобильная версия
- Публикация агента и управление доступом к нему
- Параллельный запуск нескольких агентов
- Импорт / экспорт конфигурации агента
- Комментарии и совместное редактирование
