# Спецификация: Редактор ИИ-агентов (Agent Editor)
 
## 1. Обзор
 
Редактор агентов — это визуальный canvas-интерфейс для конфигурирования ИИ-агентов. Пользователь видит граф взаимосвязанных блоков, каждый из которых отвечает за отдельный аспект агента: инструкции, расписание запусков, каналы коммуникации, инструменты, под-агентов и навыки.
 
---
 
## 2. Интеграция в рабочее место
 
### 2.1 Точка входа
 
Редактор агента открывается прямо внутри рабочего места (workspace), не уводя пользователя на отдельную страницу.
 
При переходе в режим редактирования агента:
 
- В топбаре появляется хлебная крошка: `[Иконка рабочей области] Закупки / Название агента`
- Рядом с названием агента — бейдж `Режим редактирования` (или аналогичный лейбл), чтобы пользователь понимал, что он редактирует конфигурацию, а не работает с задачей.
 
### 2.2 Layout страницы в режиме редактирования
 
```
┌──────────────────────────────────────────────────────────────┐
│ [🏷] Закупки / Название агента   [Режим редактирования]      │  ← Topbar
├─────────────────┬────────────────────────────────────────────┤
│                 │  [Чат]  [Редактор агента]                  │  ← Табы
│  ЛЕВАЯ ПАНЕЛЬ   │                                            │
│                 │                                            │
│  Исходные       │  (содержимое активного таба)               │
│  данные         │                                            │
│                 │                                            │
│  Знания         │                                            │
│  компании       │                                            │
│                 │                                            │
└─────────────────┴────────────────────────────────────────────┘
```
 
Левая панель (документы и знания) не изменяется — работает точно так же, как в обычном режиме задачи.
 
Правая часть экрана занята панелью с двумя табами.
 
### 2.3 Таб «Чат»
 
Стандартный интерфейс чата, как в обычном режиме задачи. Служит для тестирования агента: пользователь может отправить сообщение и увидеть, как агент отвечает с текущей конфигурацией. Документы из левой панели доступны в контексте так же, как обычно.
 
### 2.4 Таб «Редактор агента»
 
Занимает всю оставшуюся ширину экрана (за вычетом левой панели). Содержит полноценный canvas с блоками и соединительными линиями, описанный в разделах 3–5.
 
Поскольку canvas получает почти весь экран по горизонтали, блоки располагаются по стандартной схеме (см. раздел 2.6) без адаптивных сжатий.
 
### 2.5 Переключение между табами
 
Переключение мгновенное, состояние каждого таба сохраняется: если пользователь написал что-то в чате, а потом перешёл в редактор и вернулся — чат остаётся на том же месте.
 
### 2.6 Концепция canvas
 
Canvas — бесконечная рабочая область с блоками-карточками, соединёнными линиями. Поддерживает drag-and-drop. Блоки не привязаны к сетке, но при первом открытии располагаются по умолчанию согласно схеме ниже.
 
### 2.7 Схема расположения блоков по умолчанию
 
```
[SCHEDULE]          [AGENT]               [TOOLBOX]
                    [INSTRUCTIONS]
[CHANNELS]                                [SUB-AGENTS]
 
                                          [SKILLS]
```
 
Блоки слева (Schedule, Channels) соединены пунктирными линиями с центральным блоком Agent/Instructions. Правые блоки (Toolbox, Sub-agents, Skills) также соединены с Agent пунктирными линиями. Каждый тип связи имеет свой цвет:
 
| Связь | Цвет линии |
|---|---|
| Schedule → Agent | Оранжевый/янтарный |
| Channels → Agent | Оранжевый/янтарный |
| Agent → Toolbox | Синий/фиолетовый (пунктир) |
| Agent → Sub-agents | Зелёный (пунктир) |
| Agent → Skills | Зелёный (пунктир) |

## 3. Логика редактирования
 
### 3.1 Единый попап
 
Все операции редактирования и добавления открываются в одном унифицированном попапе, наложенном поверх canvas. Попап закрывается по клику на крестик, по `Escape` или по клику на оверлей за его пределами.
 
Попап существует в двух режимах в зависимости от типа редактируемых данных.
 
---
 
### 3.2 Режим: текстовый редактор
 
Используется когда пользователь редактирует свободный текст.
 
**Применяется для:** Instructions, создания нового Skill.
 
**Структура попапа:**
 
```
┌─────────────────────────────────────────┐
│ [Заголовок попапа]              [✕]     │
├─────────────────────────────────────────┤
│                                         │
│  [Markdown-редактор, ~70% высоты]       │
│                                         │
├─────────────────────────────────────────┤
│                        [Cancel] [Save]  │
└─────────────────────────────────────────┘
```
 
Markdown-редактор поддерживает базовую разметку (заголовки, списки, жирный, курсив, код) и вставку переменных в формате `{{variable_name}}` с автодополнением.
 
---
 
### 3.3 Режим: выбор из каталога
 
Используется когда пользователь добавляет один из предустановленных вариантов.
 
**Применяется для:** Toolbox (+ Add), Channels (+ Add channel), Sub-agents (+ Add), Schedule (+ Add).
 
**Структура попапа:**
 
```
┌────────────────────────────────────────────────────────┐
│ [Заголовок попапа]   [🔍 Поиск]              [✕]      │
├──────────────────┬─────────────────────────────────────┤
│                  │  [Плитка]  [Плитка]  [Плитка]       │
│  Категория 1  ●  │                                     │
│  Категория 2     │  [Плитка]  [Плитка]                 │
│  Категория 3     │                                     │
│  Категория 4     │                                     │
│                  │                                     │
└──────────────────┴─────────────────────────────────────┘
```
 
**Левая колонка** — вертикальный список одноуровневых категорий. Активная категория выделена. Клик по категории фильтрует плитки справа.
 
**Правая область** — сетка плиток. Каждая плитка содержит иконку, название и короткое описание (1–2 строки). Клик по плитке немедленно добавляет элемент и закрывает попап.
 
**Поиск** — фильтрует плитки по названию и описанию по всем категориям одновременно; при активном поиске левая колонка скрывается.
 
---
 
### 3.4 Маппинг: блок → режим попапа
 
| Блок | Действие | Режим попапа |
|---|---|---|
| Instructions | Edit | Текстовый редактор |
| Skills | Create | Текстовый редактор (поле Name + markdown) |
| Skills | + Add | Каталог (существующие скиллы) |
| Toolbox | + Add | Каталог (инструменты по категориям) |
| Channels | + Add channel | Каталог (типы каналов) |
| Sub-agents | + Add | Каталог (существующие агенты) |
| Schedule | + Add | Каталог (Cron / Interval / One-time) |
 
Для Schedule: после выбора плитки с типом расписания, в том же попапе ниже раскрываются поля ввода параметров (cron-строка, интервал и т.д.) + кнопка `Save`.

## 4. Компоненты (блоки)
 
Каждый блок — это карточка с:
 
- **Заголовком** (uppercase, серый текст) — название секции.
- **Действиями** (кнопки `+ Add`, `Edit`, `Create`, `MCP`) — выравнены по правому краю заголовка.
- **Телом карточки** — содержимое или плейсхолдер.
- **Пунктом подключения** (connection point) — маленький чёрный кружок на краю карточки, от которого исходит соединительная линия.
 
### 4.1 Блок AGENT
 
**Расположение:** центр canvas.
 
**Состав:** название агента (крупный жирный текст) + краткое описание (серый текст, обрезается с `...`).
 
---
 
### 4.2 Блок INSTRUCTIONS
 
**Расположение:** под блоком Agent, визуально примыкает.
 
**Заголовок:** `INSTRUCTIONS` + кнопка `✏️ Edit`.
 
**Тело:** название инструкции жирным + первые строки текста, обрезанные с `...`.
 
**Редактирование:** кнопка `Edit` → попап в режиме текстового редактора.
 
---
 
### 4.3 Блок SCHEDULE
 
**Расположение:** слева сверху.
 
**Заголовок:** `SCHEDULE` + кнопка `+ Add`.
 
**Состояния:** пустое — `No schedules configured`; заполненное — список расписаний с описанием и кнопкой удаления.
 
**Добавление:** кнопка `+ Add` → попап в режиме каталога. После выбора типа (Cron / Interval / One-time) в том же попапе раскрываются поля ввода параметров + кнопка `Save`.
 
---
 
### 4.4 Блок CHANNELS
 
**Расположение:** слева снизу.
 
**Заголовок:** `CHANNELS` + счётчик подключённых каналов.
 
**Состояния:**
- *Без идентификации:* плейсхолдер + кнопка `⚠️ Set identity`. Кнопка `+ Add channel` недоступна.
- *С идентификацией, без каналов:* `No channels configured` + кнопка `+ Add channel`.
- *С каналами:* список каналов (иконка, название, статус).
 
**Добавление:** кнопка `+ Add channel` → попап в режиме каталога. Категории: `Messaging`, `Web`, `Email`.
 
---
 
### 4.5 Блок TOOLBOX
 
**Расположение:** справа сверху.
 
**Заголовок:** `TOOLBOX` + кнопка `+ Add` + кнопка `MCP`.
 
**Тело:** список инструментов. Каждый — строка с иконкой, названием, источником, бейджем статуса (`API Key Missing` / `Active` / `Error`) и кнопками `▷` (тест) и `🗑` (удалить).
 
**Добавление:** кнопка `+ Add` → попап в режиме каталога. Категории: `Search`, `Data`, `Communication`, `Code` и др.
 
**MCP:** кнопка `MCP` → отдельный попап с формой подключения MCP-сервера (URL, auth).
 
---
 
### 4.6 Блок SUB-AGENTS
 
**Расположение:** справа по центру.
 
**Заголовок:** `SUB-AGENTS` + кнопка `+ Add`.
 
**Состояния:** пустое — `No sub-agents configured`; заполненное — список агентов с именем, статусом и кнопкой удаления.
 
**Добавление:** кнопка `+ Add` → попап в режиме каталога. Категории: `My agents`, `Team agents`.
 
---
 
### 4.7 Блок SKILLS
 
**Расположение:** справа снизу.
 
**Заголовок:** `SKILLS` + кнопка `+ Add` + кнопка `✏️ Create`.
 
**Состояния:** пустое — `No skills configured`; заполненное — список навыков с именем, кратким описанием и кнопкой удаления.
 
**Добавление существующего:** кнопка `+ Add` → попап в режиме каталога.
 
**Создание нового:** кнопка `Create` → попап в режиме текстового редактора. Поле `Name` (однострочный input) над markdown-редактором.

## 5. Соединительные линии
 
Линии визуализируют зависимости между блоками.
 
| Свойство | Значение |
|---|---|
| Стиль | Пунктирная кривая (bezier curve) |
| Цвет | Зависит от типа связи (см. раздел 2.2) |
| Точки подключения | Чёрные кружки (`●`) на краях карточек |
| Поведение при перемещении блока | Линия перерисовывается динамически |
 
**Логика прокладки линий:**
- Линии огибают блоки, не пересекают их.
- При слишком малом расстоянии между блоками линия уходит в дугу, увеличивая радиус кривизны.
 
---
 
## 6. Состояния блоков
 
| Состояние | Визуальное отображение |
|---|---|
| Пустое / не настроено | Серый плейсхолдер-текст внутри карточки |
| Требует действия | Жёлтая/оранжевая кнопка с иконкой `⚠️` |
| Ошибка | Красный бейдж или красная граница карточки |
| Активно / OK | Стандартный вид, без дополнительных индикаторов |
| Загрузка | Скелетон-анимация внутри тела карточки |
 
---
 
## 7. Интерактивность canvas
 
| Действие | Поведение |
|---|---|
| Drag блока | Блок перемещается, линии перерисовываются |
| Scroll / pinch | Зум canvas (min: 50%, max: 200%) |
| Правый клик по canvas | Контекстное меню: `Add block`, `Paste`, `Zoom to fit` |
| Правый клик по блоку | `Duplicate`, `Delete`, `Collapse/Expand` |
| Двойной клик по пустому месту | Создать новый блок (dropdown: тип блока) |
| `Ctrl/Cmd + Z` | Undo |
| `Ctrl/Cmd + Shift + Z` | Redo |
 
---
 
## 8. Адаптивность
 
- **Desktop (≥1024px):** полноценный canvas с горизонтальным расположением блоков.
- **Tablet (768–1023px):** блоки расположены в 2 колонки, canvas скроллится вертикально.
- **Mobile (<768px):** блоки отображаются как вертикальный стек-аккордеон без canvas-графики.
 
---
 
## 9. Сохранение состояния
 
- Изменения в блоках сохраняются автоматически (auto-save) с задержкой 1–2 секунды после последнего действия.
- В topbar отображается статус: `Saving...` → `Saved` → без текста.
- Поддерживается версионирование конфигурации: кнопка `History` в топбаре открывает список снапшотов.
