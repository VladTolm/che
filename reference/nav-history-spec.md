# Спецификация: Навигация и layout пространства
**Версия:** 2.0  
**Статус:** Готово к реализации

---

## 1. Принципы

**Три режима работы с пространством** — каждый отвечает на свой вопрос:

| Пункт | Вопрос | Назначение |
|---|---|---|
| + Новая задача | Что сделать? | Создать сессию и работать с агентом |
| История | Что было сделано? | Просматривать сессии и управлять ими |
| Пространство | Как устроено? | Настраивать контекст, агентов, участников |

**Три уровня абстракции разделены:**
- *Агент* — окружение сессии, выбирается на стартовом экране, не меняется в рамках сессии.
- *Инструменты и навыки* — оперативные настройки внутри сессии, доступны через поле ввода.
- *Сессии* — задачи с атрибутами, живут на странице «История».

**Лево = навигация, право = контекст текущей страницы.**

## 2. Левая боковая панель

### 2.1 Структура

```
┌──────────────────────────┐
│  🛍 Закупки              │  ← шапка пространства
├──────────────────────────┤
│  + Новая задача          │
│    История          ●2   │  ← бейдж = сессии в «Ожидание»
│    Пространство          │
└──────────────────────────┘
```

Три фиксированных пункта под каждым пространством. Никаких списков сессий или документов в навигации.

### 2.2 Поведение пунктов

**«+ Новая задача»** — открывает стартовый экран с пустым полем и выбором агента. Текущая открытая сессия сохраняется и остаётся доступной через «История».

**«История»** — открывает страницу истории сессий. Бейдж `●N` — количество сессий в статусе «Ожидание». При `N = 0` бейдж скрыт.

**«Пространство»** — открывает страницу управления пространством: документы, агенты, участники, настройки доступа.

## 3. Стартовый экран (новая сессия)

### 3.1 Layout

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│                   [Иконка агента]                        │
│                 Финансовый аналитик                      │
│         Анализ договоров, план-факт, отчётность          │
│                                                          │
│   ┌──────────────────────────────────────────────────┐   │
│   │  Чем я могу помочь сегодня?                      │   │
│   │                                                  │   │
│   │  [📁]  [/]                        [⚙]  [📅]  [→]│   │
│   └──────────────────────────────────────────────────┘   │
│                                                          │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ ✦ Универсальный  🔍 Ресёрчер  📝 Документалист  ›  │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 3.2 Центральная зона

Показывает текущего выбранного агента — иконку, название, описание. По умолчанию универсальный агент. Обновляется при выборе другого агента (анимация fade).

### 3.3 Поле ввода

| Элемент | Описание |
|---|---|
| `📁` | Прикрепить документы сессии (IN) |
| `/` | Вызвать навык — открывает выпадающий список с двумя секциями: навыки компании (`/`) и навыки пространства (`./`) |
| `⚙` | Настройки сессии — инструменты и параметры |
| `📅` | Расписание или триггер для сессии |
| `→` | Отправить и начать сессию |

Плейсхолдер: «Чем я могу помочь сегодня?» для универсального агента, «Опишите задачу для [Агент]...» для преднастроенного.

### 3.4 Переключатель агентов

Горизонтальная полоса под полем ввода. Активный агент подсвечен. При переполнении — горизонтальный скролл с индикатором `›`. После отправки первого сообщения полоса исчезает.

## 4. Layout активной сессии

### 4.1 Структура

```
┌────────────────────────────────────────────────────────────────┐
│ [←]  🛍 Закупки  /  Бриф ФосАгро Q2     [📝 Документалист]   │  ← топбар
├──────────────────────────────────────┬─────────────────────────┤
│                                      │  Исполнение             │
│                                      │  ✓ Шаг 1               │
│              ЧАТ                     │  ✓ Шаг 2               │
│                                      │  ⟳ Шаг 3               │
│                                      │  ○ Шаг 4               │
│                                      │                         │
│                                      │  Инструменты            │
│                                      │  [Битрикс24]  CRM       │
│                                      │  [1С:ERP]     ERP       │
│                                      │                         │
│   ┌───────────────────────────────┐  │  Навыки                 │
│   │  Ответить агенту...           │  │  /summarize             │
│   │  [📁]  [/]        [⚙][📅][→] │  │  /extract               │
│   └───────────────────────────────┘  │                         │
└──────────────────────────────────────┴─────────────────────────┘
```

### 4.2 Топбар

- `[←]` — возврат к стартовому экрану или «Истории».
- Хлебная крошка: `[Пространство] / [Название сессии]`.
- Бейдж агента справа — иконка и название. Клик открывает карточку агента (только просмотр, не смена).

### 4.3 Правая панель

**Исполнение** — отображается когда агент выполняет задачу. Шаги с иконками: `✓` выполнен, `⟳` в процессе, `○` ожидает. В режиме свободного чата скрыта.

**Инструменты** — инструменты агента. Бейдж `⚠ Требует настройки` если пользователь не авторизован в инструменте.

**Навыки** — навыки агента и пространства. Клик подставляет `/команду` в поле ввода.

### 4.4 Состояния поля ввода

| Статус сессии | Состояние поля |
|---|---|
| Активна | Редактируемое |
| Ожидание | Редактируемое, плейсхолдер «Ответьте агенту или нажмите Продолжить» |
| Исполнение | Заблокировано, кнопка `→` → `⏹ Остановить` |
| Завершена / Архив | Заблокировано, баннер «Сессия завершена» |

## 5. Страница «История»

### 5.1 Layout

Двухколоночный. Левая — фильтры и список. Правая — детали выбранной сессии.

```
┌────────────────────────────────────────────────────────────────┐
│  🛍 Закупки  /  История                                        │
├───────────────────────┬────────────────────────────────────────┤
│  [Поиск...        🔍] │                                        │
├───────────────────────┤  (детали выбранной сессии)             │
│  Статус               │                                        │
│  □ Ожидание      ●2   │                                        │
│  □ Активна            │                                        │
│  □ Завершена          │                                        │
│  □ Архив              │                                        │
│  Агент  [Все ▾]       │                                        │
│  Участник [Все ▾]     │                                        │
│  Период [с] — [по]    │                                        │

│  Ожидание             │                                        │
│  ────────────────     │                                        │
│  🔍 Ресёрчер          │                                        │
│  Входящий запрос...   │                                        │
│  09:45       [Взять]  │                                        │
│                       │                                        │
│  ⚖ Юрист              │                                        │

│                       │                                        │
```

### 5.2 Список сессий

Группировка: «Ожидание» всегда вверху, «Остальные» — по дате убывания.

Каждая строка: иконка агента, название сессии, превью задачи, время, статус-бейдж, владелец (в режиме «Общие сессии»), кнопка «Взять» для сессий «Ожидание».

### 5.3 Детали сессии

```
┌──────────────────────────────────────────────────┐
│ Бриф ФосАгро Q2               [Открыть]  [···]  │
│ 📝 Документалист · Влад · 14 янв, 14:32          │
├──────────────────────────────────────────────────┤
│ СТАТУС      Завершена                            │
│ АГЕНТ       Документалист                        │

│ СОЗДАНА     14 янв 2025, 14:32                   │
│ ЗАВЕРШЕНА   14 янв 2025, 16:05                   │
├──────────────────────────────────────────────────┤
│ ДОКУМЕНТЫ СЕССИИ                                 │
│  [PDF] Бриф_ФосАгро_Q2.pdf            IN        │
│  [MD]  Анализ_рынка.md                OUT  [👁] │
├──────────────────────────────────────────────────┤
│ ИСТОРИЯ ПЕРЕНАЗНАЧЕНИЙ                           │
│  Анна → Влад · 14 янв, 13:15                    │
├──────────────────────────────────────────────────┤
│ ПРЕДПРОСМОТР                                     │
│  Пользователь: Проанализируй бриф...             │
│  Агент: На основе документа...                   │
│                          [Показать полностью]    │
└──────────────────────────────────────────────────┘
```

**Кнопка «Открыть»:**

| Статус | Поведение |
|---|---|
| Активна, текущий пользователь — владелец | Рабочий режим |
| Ожидание без владельца | Модальное окно принятия |
| Ожидание, чужой владелец | Режим чтения |
| Завершена / Архив | Режим чтения, баннер «Сессия завершена» |

**Меню `···`:** «Перевести в архив» (для завершённых), «Переназначить» (для «Ожидание» в режиме «Общие сессии»).

## 6. Страница «Пространство»

Страница управления контекстом и настройками пространства. Состоит из трёх секций, каждая занимает полную ширину и следует одна за другой.

### 6.1 Layout

```
┌────────────────────────────────────────────────────────────────┐
│  🛍 Закупки  /  Пространство                                   │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ДОКУМЕНТЫ                                           [+ Добавить]
│  ──────────────────────────────────────────────────────────── │
│  [PDF] Политика_закупок.pdf              14 янв      [🗑]     │
│  [DOCX] Шаблон_договора.docx             12 янв      [🗑]     │
│  [MD]  Инструкция_по_работе.md           10 янв      [🗑]     │
│                                                                │
│  Эти документы автоматически попадают в контекст              │
│  каждой новой сессии в пространстве                           │
│                                                                │
│  АГЕНТЫ                                            [+ Создать] │
│  ──────────────────────────────────────────────────────────── │
│  [🔍] Ресёрчер                                                 │
│       Поиск и анализ информации из документов    [Настроить]  │
│                                                                │
│  [📝] Документалист                                            │
│       Формирование отчётов и документов          [Настроить]  │
│                                                                │
│  ДОСТУП                                                        │
│  ──────────────────────────────────────────────────────────── │
│  Тип доступа                                                   │
│  ● Общие сессии                                   [Изменить]  │
│                                                                │
│  Участники                                    [+ Пригласить]  │
│  [В] Влад Иванов          Владелец                            │
│  [А] Анна Смирнова        Администратор                   ▾   │
│  [П] Игорь Петров         Участник                        ▾   │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 6.2 Секция «Документы»

Документы пространства — файлы, которые автоматически попадают в контекст агента при старте каждой сессии.

| Элемент | Описание |
|---|---|
| Кнопка «+ Добавить» | Открывает файловый диалог. Поддерживаемые форматы: PDF, DOCX, TXT, MD, CSV |
| Строка документа | Иконка типа, название, дата загрузки, кнопка удаления `🗑` |
| Drag & drop | Файл можно перетащить в область секции |
| Пустое состояние | «Нет документов. Добавьте файлы, которые агент будет использовать в каждой сессии» |

Подпись под списком: «Эти документы автоматически попадают в контекст каждой новой сессии в пространстве».

### 6.3 Секция «Агенты»

Список преднастроенных агентов пространства.

| Элемент | Описание |
|---|---|
| Кнопка «+ Создать» | Открывает редактор агента (agent editor) |
| Строка агента | Иконка, название, описание, кнопка «Настроить» |
| Кнопка «Настроить» | Открывает редактор этого агента |
| Пустое состояние | «Нет агентов. Создайте первого агента для пространства» + кнопка «+ Создать» |

Универсальный агент в этом списке не отображается — он всегда доступен по умолчанию.

### 6.4 Секция «Доступ»

**Тип доступа** — текущий тип отображается текстом с кнопкой «Изменить» справа. Клик на «Изменить» — разворачивает радиокнопки для выбора:

```
  ○  Персональный
     Видно только вам

  ○  Общий контекст
     Участники видят агентов и документы. Сессии у каждого свои

  ●  Общие сессии
     Всё общее, включая сессии участников

              [Отмена]  [Сохранить]
```

Доступно только владельцу. Администратор видит текущий тип без кнопки «Изменить».

При сохранении понижающего типа — предупреждение:

```
┌────────────────────────────────────────────────┐
│  ⚠️  Изменение типа доступа                    │
│                                                │
│  Сессии других участников перейдут в режим    │
│  «только чтение». Переназначить их можно       │
│  в разделе «История».                          │
│                                                │
│              [Отмена]  [Изменить тип]          │
└────────────────────────────────────────────────┘
```

**Участники** — список с ролями. Отображается только при типе «Общий контекст» и «Общие сессии».

| Элемент | Описание |
|---|---|
| Строка участника | Аватар, имя, дропдаун роли |
| Дропдаун (для владельца) | `Администратор` / `Участник` / `— Удалить из пространства` |
| Права администратора | Может менять роли участников, не может менять роли других администраторов и владельца |
| Кнопка «+ Пригласить» | Поиск по имени или email, новый участник получает роль «Участник» |

## 7. Пустые состояния

| Страница | Ситуация | Отображение |
|---|---|---|
| История | Нет сессий | «Сессий пока нет» + кнопка «+ Новая задача» |
| История | Фильтры без результатов | «Ничего не найдено» + «Сбросить фильтры» |
| История | Ничего не выбрано в правой колонке | «Выберите сессию для просмотра» |
| Пространство / Документы | Нет документов | «Добавьте файлы, которые агент будет использовать в каждой сессии» |
| Пространство / Агенты | Нет агентов | «Создайте первого агента» + кнопка «+ Создать» |
