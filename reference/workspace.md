# Рабочее пространство — UI Спецификация v1.0
 
---
 
## 1. Обзор и цели
 
Рабочее пространство — ключевой раздел платформы для работы с ИИ-агентами. Обеспечивает два режима работы: свободный чат (Q&A) и агентный режим с циклом **планирование → верификация плана → исполнение → верификация результата**.
 
**Цели функциональности:**
- Работа с загруженными документами пользователя (Исходные данные)
- Работа со структурированными знаниями компании (топики, папки, файлы)
- Режим Q&A: свободный диалог с выбранным контекстом
- Агентный режим: предустановленный контекст, пошаговое исполнение, верификация
 
---
 
## 2. Общая структура интерфейса
 
Экран разделён на три вертикальные колонки. Левая и правая панели сворачиваются, освобождая пространство под чат.
 
| Зона | Ширина | Описание |
|---|---|---|
| Левая панель | ~280px | Исходные данные + Знания компании |
| Центральная область | flex | Чат. Растягивается при скрытых боковых панелях |
| Правая панель | ~280px | Навыки, агенты, исполнение, артефакты |
 
**Поведение при сворачивании:**
- Иконка-стрелка на границе панели. Нажатие скрывает панель; граница остаётся кликабельной для раскрытия
- Состояние сворачивания сохраняется в localStorage
- При одновременно свёрнутых обеих панелях чат занимает всю ширину

## 3. Левая панель
 
### 3.1 Исходные данные
 
Раздел для загрузки документов, с которыми пользователь работает в текущей сессии. Результаты работы агентов также могут сохраняться сюда.
 
**Элементы:**
- Заголовок «Исходные данные» + кнопка «+» для загрузки
- Список документов: иконка типа файла, название, кнопка удаления (по hover)
- Чекбокс рядом с каждым документом — выбранные попадают в контекст чата
- Пустое состояние: «Загрузите документы для работы» + кнопка «Загрузить»
 
**Взаимодействие:**
- Drag & drop файлов в область раздела
- Клик по «+» открывает стандартный файловый диалог
- Поддерживаемые форматы: PDF, DOCX, TXT, MD, CSV
- После сохранения артефакта агента — файл появляется в списке с тегом «Артефакт»
 
### 3.2 Знания компании
 
Структурированные знания из внешних корпоративных источников. Аналог git-репозитория: каждый топик содержит вложенную структуру папок и файлов. Пользователь видит только топики, к которым у него есть доступ.
 
**Элементы:**
- Список топиков — сворачиваемые узлы верхнего уровня
- Внутри топика: древовидная структура папок и файлов
- Чекбоксы на каждом уровне (топик / папка / файл)
- Пустое состояние: «Нет доступных топиков»
 
**Выбор контекста:**
- Выбор топика целиком — добавляет всё содержимое
- Выбор папки — добавляет все файлы папки
- Выбор отдельных файлов
 
**Индикатор контекста:**
- В нижней части левой панели — счётчик: «В контексте: 3 документа, 2 файла знаний»

## 4. Центральная область — Чат
 
### 4.1 Режим Q&A (Freeride)
 
Режим по умолчанию. Пользователь общается свободно. Контекст формируется из выбранных в левой панели документов и знаний.
 
**Поле ввода:**
- Многострочный textarea с автовысотой
- Теги выбранных источников над полем ввода (кликабельны для снятия выбора)
- Ввод «/» открывает выпадающий список доступных навыков
- Кнопка «Отправить» (→) активна при наличии текста
 
**История сообщений:**
- Сообщения пользователя — выровнены вправо, пузырь с фоном
- Ответы модели — выровнены влево, маркдаун-рендеринг
- Кнопки под каждым ответом: «Скопировать», «Сохранить как артефакт»
 
### 4.2 Режим агента
 
Активируется выбором агента в правой панели. Контекст зафиксирован агентом — пользователь не может его изменить. Левая панель переходит в режим «только чтение» с отображением контекста агента.
 
**Цикл выполнения:**
 
1. **Планирование** — агент показывает план в виде нумерованного списка шагов
2. **Верификация плана** — в чате появляется карточка подтверждения с кнопками «Запустить» и «Отменить»
3. **Исполнение** — поле ввода заблокировано; прогресс отображается в правой панели (вкладка «Исполнение»)
4. **Верификация результата** — агент сообщает о завершении, предлагает принять результат или запросить доработку
 
**Карточка верификации плана:**
 
| Элемент | Описание |
|---|---|
| Содержимое | Краткое описание задачи + список шагов плана |
| Кнопка «Запустить» | Запускает исполнение, блокирует ввод |
| Кнопка «Отменить» | Возвращает в режим ввода без сброса контекста |
 
**Блокировка ввода во время исполнения:**
- Textarea неактивна с плейсхолдером «Агент работает...»
- Кнопка «Остановить» вместо кнопки отправки — прерывает выполнение
- После остановки: агент сообщает что было выполнено, предлагает продолжить или сбросить

## 5. Правая панель
 
Содержит три вкладки. Активная вкладка переключается автоматически по контексту, либо вручную.
 
### 5.1 Вкладка «Агенты и навыки»
 
Отображается по умолчанию.
 
**Секция «Режим работы»:**
- Пункт «Свободный режим» — возвращает в freeride, снимает активного агента
- Список агентов: иконка, название, краткое описание, кнопка «Выбрать»
- Активный агент подсвечен, рядом — индикатор «активен»
- Пустое состояние: «Нет доступных агентов — обратитесь к администратору»
 
**Секция «Навыки»:**
- Список навыков с описанием — при клике подставляет слеш-команду в поле чата
- Поиск по навыкам
- Пустое состояние: «Навыки не настроены»
 
### 5.2 Вкладка «Исполнение»
 
Становится активной автоматически при запуске агента.
 
| Состояние | Триггер | Что видит пользователь |
|---|---|---|
| Ожидание | Агент выбран, задача не запущена | Заглушка: «Запустите задачу в чате» |
| Планирование | Агент формирует план | Спиннер + «Агент формирует план...» |
| Верификация плана | Ожидает подтверждения | Список шагов, статус «Ожидает подтверждения» |
| Выполнение | Задача запущена | Шаги с иконками: ✓ выполнен / ⟳ в процессе / ○ ожидает |
| Верификация результата | Агент завершил | Все шаги ✓, кнопка «Принять результат» |
| Остановлено | Пользователь нажал «Стоп» | Прогресс до момента остановки, кнопка «Продолжить» |
 
### 5.3 Вкладка «Артефакт»
 
Появляется и становится активной автоматически когда агент генерирует артефакт.
 
**Элементы:**
- Превью артефакта (рендер маркдауна, таблицы или текста)
- Автоматически сформированное название (на основе задачи и даты)
- Кнопка «Сохранить в Исходные данные» — добавляет файл с тегом «Артефакт»
- Кнопка «Сохранить в Знания компании» — открывает выбор топика (если есть права на запись)
- Кнопка «Скачать»

## 6. Состояния и пустые экраны
 
| Состояние | Триггер | Что видит пользователь |
|---|---|---|
| Новый пользователь | Первый вход, нет данных | Обе панели показывают онбординг-подсказки с призывами к действию |
| Нет документов | Раздел «Исходные данные» пуст | Иллюстрация + «Загрузите документ для начала работы» + кнопка |
| Нет знаний | Нет доступных топиков | «Топики не настроены или нет доступа» |
| Нет агентов | Список агентов пуст | «Агенты не настроены — обратитесь к администратору» |
| Загрузка документа | Файл индексируется | Прогресс-бар в списке, файл недоступен для выбора |
| Ошибка агента | Агент не смог выполнить шаг | Сообщение в чате + детали во вкладке «Исполнение», кнопка «Повторить шаг» |
 
---
 
## 7. Слеш-команды и навыки
 
Навыки — шаблоны поведения, активируемые через «/» в чате. В отличие от агентов, навыки не несут собственного контекста.
 
**Поведение:**
- Ввод «/» в поле чата: выпадающий список навыков с иконкой и кратким описанием
- Ввод символов после «/»: фильтрация по названию
- Выбор навыка: подставляет системный промпт / шаблон в сообщение
- Навык не блокирует ввод и не переключает правую панель
 
---
 
## 8. Вне области текущей спецификации
 
- Мобильная версия
- История сессий (доступна на уровень выше в навигации)
- Параллельный запуск нескольких агентов
- Редактирование плана агента до запуска
